# 사유 인프라/비용 브레인스토밍

## 1. 목적
이 문서는 사유 플랫폼의 초기 제작 인프라와 운영 비용을 미확정 아이디어 중심으로 정리한다.
확정 전 가설, 선택지, 비용 범위를 빠르게 비교하는 용도로 사용한다.

## 2. 전제 조건
- 1인 또는 소수 인원으로 시작
- 웹 + 앱 동시 운영을 목표로 하되, 초기에는 공통 백엔드 우선
- 빠른 출시와 운영 단순성을 우선
- 과투자보다 단계적 확장 전략 채택

## 3. 아키텍처 옵션 (초기)
### 옵션 A. Managed 중심 (권장)
- 프론트: 웹 정적 배포 + 앱 클라이언트
- 백엔드: BaaS 또는 Managed DB + Auth + Storage
- 장점: 빠른 개발, 운영 부담 낮음
- 단점: 트래픽 증가 시 비용 변동성

### 옵션 B. 자체 서버 중심
- 프론트/백엔드 모두 자체 서버(VM/Container) 운영
- 장점: 구조 커스터마이징 자유도 높음
- 단점: DevOps 부담 큼, 장애 대응 책임 증가

### 옵션 C. 혼합형
- 인증/DB/스토리지는 Managed, 일부 API는 자체 서버
- 장점: 속도와 유연성 균형
- 단점: 설계 복잡도 증가

## 4. 초기 권장 인프라 구성안 (가설)
- 웹: CDN 기반 정적 호스팅
- 앱: React Native 또는 Flutter 중 1개 선택
- 인증: 이메일/소셜 로그인 지원 가능한 Managed Auth
- 데이터베이스: Managed Postgres 계열
- 파일 저장소: 이미지/첨부 중심 Object Storage
- 검색: 초기엔 DB 기반 검색, 추후 검색엔진 도입
- 알림: 앱 푸시 + 이메일(필수 이벤트만)
- 로그/모니터링: 에러 추적 + 성능 모니터링 최소 구성

## 5. 비용 항목 체크리스트
- 도메인/SSL
- 웹 호스팅/CDN
- DB 인스턴스
- 스토리지/트래픽
- 인증/메시징(이메일, SMS)
- 푸시 알림
- 결제 연동 수수료(PG)
- 로그/모니터링 툴
- 앱 스토어 등록 비용(Apple/Google)
- 법무/약관/개인정보 정책 준비 비용

## 6. 월 비용 러프 범위 (미확정)
정확 견적이 아닌 의사결정용 범위이다.

- Stage 0 (개발/클로즈드 테스트): 월 5만원 ~ 20만원
- Stage 1 (초기 오픈, 사용자 소수): 월 20만원 ~ 80만원
- Stage 2 (활성 사용자 증가): 월 80만원 ~ 300만원+

비용 급증 포인트:
- 이미지/파일 트래픽
- 검색 고도화
- 알림 대량 발송
- 결제/정산 운영 자동화

## 7. 결제 도입 시 추가 인프라 포인트
- 결제 상태 웹훅 처리 안정성
- 중복 결제/실패 재시도 처리
- 정산 배치 및 리포트 생성
- 환불/분쟁 이력 추적
- 감사 로그(누가 언제 어떤 정산을 수행했는지)

## 8. 인프라 선택 기준 (의사결정 질문)
- 지금 필요한 것은 속도인가, 커스터마이징인가?
- 장애 대응을 누가 할 것인가?
- 월 고정비와 트래픽 연동비 중 무엇을 선호하는가?
- 앱과 웹의 배포 주기를 어떻게 맞출 것인가?
- 유료 기능 도입 시 정산 투명성을 어디까지 자동화할 것인가?

## 9. 리스크 메모
- 초기 과설계: 출시 전 기술 선택 과도화
- 벤더 종속: 특정 서비스 의존도 과다
- 운영 공백: 로그/알림 부재로 장애 감지 지연
- 정책 지연: 결제/환불 정책 미정으로 CS 폭증

## 10. 다음 대화에서 결정할 것
1. 아키텍처 옵션 A/B/C 중 1차 선택
2. 앱 기술 스택 1차 선택(React Native vs Flutter)
3. 초기 월 예산 상한선 확정
4. 결제 도입 시점(출시 동시 vs 후속)
5. 필수 모니터링 항목 5개 정의

## 11. 현재 조건 기준 비용 추정 (사용자 입력 반영)
조건:
- MVP는 결제 기능 없이 운영
- 앱 광고(AdMob 등)만 우선 도입
- DB는 클라우드 서비스 이용
- 제작/운영은 1인 + AI 협업

### 11.1 초기 1회성 비용
- 도메인: 연 2만원 ~ 5만원
- Google Play 개발자 등록: 약 25 USD (약 3.5만원, 1회)
- Apple Developer Program: 연 99 USD (약 14만원)
- 앱 출시용 디자인/스토어 리소스: 0원 ~ 30만원
- 약관/개인정보 문서 정비: 0원 ~ 50만원 (직접 작성 시 0에 가까움)

정리:
- 안드로이드만 먼저 출시하면 초기 현금 지출은 대체로 10만원 ~ 40만원
- iOS까지 동시 출시하면 첫해 기준 대체로 25만원 ~ 80만원

### 11.2 단기 월 운영비 (출시 초기, MAU 1천 이하 가정)
- 클라우드 DB/BaaS: 0원 ~ 15만원
- 스토리지 + CDN 트래픽: 1만원 ~ 8만원
- 로그/모니터링: 0원 ~ 8만원
- 이메일/알림/운영도구: 0원 ~ 5만원
- AI 도구 사용료(개발 보조): 3만원 ~ 40만원

합계(단기):
- 월 5만원 ~ 40만원 (보수적으론 60만원까지 버퍼 권장)

### 11.3 중기 월 운영비 (MAU 1천 ~ 1만)
- 인프라 + 관측성 + 운영도구 포함
- 월 30만원 ~ 180만원

비용이 튀는 구간:
- 이미지/파일 업로드와 조회 증가
- 검색/피드 고도화
- 푸시/이메일 대량 발송

### 11.4 장기 월 운영비 (MAU 1만 ~ 5만+)
- 월 150만원 ~ 700만원+

이 구간부터는:
- DB 스케일 업/리드 레플리카
- 캐시/큐/배치 분리
- 운영 자동화 비용
이 본격적으로 붙는다.

### 11.5 광고 수익 관점 메모
- 초기에는 광고 수익이 인프라비를 충분히 커버하지 못할 수 있다.
- 즉, 광고는 초반에 "비용 일부 회수" 관점으로 보는 것이 현실적이다.
- MAU와 체류시간이 안정화되면 손익분기 가능성이 올라간다.

### 11.6 실행용 예산 가이드 (권장)
1. 출시 전 6개월 버퍼: 월 40만원 기준으로 240만원 확보
2. 앱 동시 출시 버퍼: 스토어/운영 포함 추가 50만원 ~ 150만원 확보
3. 총 준비금 가이드: 최소 300만원 ~ 600만원

위 범위는 빠른 검증형 MVP 기준이다. 기능 확장 속도가 빠르면 비용 상단에 근접한다.
