# Noname 1 Poem Archive

## 1. 프로젝트 개요
이 프로젝트는 시(텍스트) 아카이브를 보여주는 정적 단일 페이지 웹사이트입니다.

- 단일 파일 구조: `index.html` 안에 HTML, CSS(Tailwind), JavaScript가 모두 포함됨
- 핵심 목적: 감성적인 시 목록 탐색, 태그 필터링, 검색, 상세 모달 보기 및 관리자 게시물 관리 제공
- 데이터 소스: Google Apps Script API (Google Sheet 기반)

## 2. 기술 스택
- HTML5
- Tailwind CSS CDN
- Vanilla JavaScript
- Google Fonts (Noto Serif KR, Noto Sans KR)
- LocalStorage (테마/시트 URL 저장)

## 3. 화면 및 기능 구성
### 3.1 헤더
- 브랜드 타이틀(무명1 / Noname 1)
- 라이트/다크 테마 토글
- Instagram 외부 링크

### 3.2 메인 영역
- 소개 문구(작가 톤앤매너 강조)
- 태그 필터 영역
- 검색 입력창(제목/본문/코멘트 검색)
- 시 아카이브 목록 렌더링

### 3.3 상세 모달
- 시 제목, 본문, 태그, 제작일시, 작가 코멘트 표시
- 오버레이 클릭 시 닫기

### 3.4 설정 모달(관리)
- Apps Script Web App URL 저장
- URL 초기화(기본 Apps Script URL 복원)

### 3.5 관리자 패널
- 관리자 로그인(클라이언트 비밀번호)
- 게시물 CRUD (생성/수정/삭제)
- 게시물 순서 변경(위/아래)
- 게시물 공개/비공개 전환
- 관리자 변경사항 Apps Script API 반영

### 3.6 토스트 알림
- 연동 상태/에러 등의 피드백을 비차단 UI로 제공

## 4. 데이터 구조
코드에서 시 데이터는 다음 한글 키 스키마를 사용합니다.

- 제목
- 본문
- 태그
- 제작일시
- 작가코멘트
- 인스타링크
- 공개여부

클라이언트는 초기 로드 시 Apps Script API에서 공개 게시물 목록을 조회합니다.

## 5. 동작 흐름
1. `DOMContentLoaded` 시점에 테마 초기화
2. LocalStorage에서 Apps Script URL 조회
3. `action=list&publishedOnly=true`로 공개 게시물 조회
4. `poemsData` 갱신 후 태그 UI 및 목록 렌더링
5. 검색어/태그 변경 시 필터 재적용
6. 카드 클릭 시 모달 오픈

## 6. CSV 파싱 로직 특징
`parseCSV` 함수는 단순 split이 아니라 상태 기반 파싱을 사용합니다.

- 따옴표 내부 쉼표 처리
- 개행 포함 셀 처리
- `""` 이스케이프 처리
- `\\n` 문자열을 실제 개행으로 치환

이 덕분에 시 본문처럼 줄바꿈이 많은 텍스트를 안정적으로 읽어올 수 있습니다.

## 7. 강점
- 단일 파일 배포로 매우 간단한 운영 구조
- 감성적인 타이포그래피/여백 중심 UI
- 실사용 기능(필터/검색/모달) 완비
- Google Sheet 연동으로 비개발자도 콘텐츠 관리 가능

## 8. 개선 포인트
1. XSS 방어: 현재 일부 내용이 `innerHTML`로 렌더링되므로 외부 입력에 대한 이스케이프 처리 권장
2. 접근성: 모달 포커스 트랩, ESC 닫기, 키보드 내비게이션 강화 권장
3. 성능: 데이터가 커질 경우 렌더링 최적화(가상 스크롤 또는 페이지네이션) 고려
4. 유지보수: JS를 별도 파일로 분리하여 관심사 분리 권장
5. 설정 보안: CSV URL 유효성 검사/도메인 제한 추가 권장

## 9. 실행 방법
정적 페이지이므로 별도 빌드 없이 `index.html`을 브라우저에서 열면 동작합니다.
